# Project Overview

## 1. Project Name
**EduSaaS** (School/Institute Management System)

## 2. Mission & Purpose
EduSaaS is a comprehensive, cloud-based platform designed to modernize the management of schools, coaching centers, and educational institutes. The goal is to replace paper-based and disjointed digital workflows with a single, unified dashboard. It aims to streamline administrative tasks, enhance communication, and provide actionable insights through analytics.

## 3. Key Objectives
- **Centralization**: Manage students, batches, attendance, assignments, and billing in one place.
- **Automation**: Automate attendance tracking and fee collection/subscription management.
- **User Experience**: Provide a "delightful", premium, and accessible UI/UX for administrators.
- **Monetization**: Built as a SaaS (Software as a Service) to be sold to multiple institutes via subscription plans (Basic overlay/Pro).

## 4. Primary Features
- **Authentication & Onboarding**: Secure signup/login via Supabase Auth.
- **Dashboard**: High-level analytics (Student count, Active batches, Attendance trends).
- **Student Management**: CRUD operations for student records.
- **Batch Management**: Organize students into batches/classes.
- **Attendance**: Daily tracking of student presence.
- **Academics**:
    - **Announcements**: Broadcast messages to batches or the entire school.
    - **Assignments**: Create tasks with file attachments (PDFs/Images).
- **Billing**: Stripe integration for SaaS subscription management (Basic vs Pro tiers).

## 5. Target Audience
- **Primary**: School Administrators, Coaching Center Owners.
- **Secondary**: Teachers (future scope), Students (future scope).

## 6. Development Philosophy
- **"Visual Excellence"**: The tool must look premium (dark mode, glassmorphism, smooth transitions).
- **"Agent-Friendly"**: The codebase is structured to be easily understood and modified by AI agents.
- **"Performance"**: Usage of Next.js Server Components and optimized database queries.
